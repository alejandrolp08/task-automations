# Product Vision

## Working Name

TCG Listing Platform

## Product Type

A web-first intake and listing preparation platform for TCG sellers.

## Initial Market

- Pokemon micro sellers
- Pokemon small sellers
- collectors selling batches
- sellers who do not want to buy expensive scanning hardware
- sellers who want a simpler intake workflow before listing on eBay

## Problem

Current listing workflows are painful for smaller sellers because they often require:
- one photo or scan per card
- expensive hardware
- too many files
- repetitive matching and correction work
- heavy tools built for more advanced sellers

## Initial Differentiator

The first differentiator is a single-video intake flow:
- one video
- one upload
- one batch
- one review flow

The product should also support multiple image uploads so users can choose the method that matches their current setup.

## Initial Value Proposition

Turn a Pokemon batch into listing-ready inventory faster, with less friction, and without requiring a dedicated scanner.

## Product Principles

1. Intake first
The first goal is converting physical cards into usable digital inventory.

2. Review by exception
Users should only spend time correcting doubtful matches.

3. Flexible capture
Video-first, but not video-only.

4. Web-first simplicity
The first version should work without a mobile app.

5. Controlled scope
Start with Pokemon, eBay CSV export, and stock catalog images.

## Out of Scope for V1

- Multi-game support
- direct eBay API listing
- TCGplayer direct publishing
- condition grading automation
- automated frame-based listing photos
- advanced repricing engine
- physical capture hardware
